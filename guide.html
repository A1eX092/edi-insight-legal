Prise en main rapide

Qu'est-ce qu'EDI Insight ?

Une application spécialisée dans les flux de paiement bancaires : virements et prélèvements SEPA (ISO 20022), comptes rendus (pacs.002, pain.002, camt), erreurs EBICS et messages EDIFACT. Elle sert à vérifier la conformité d'un fichier, comprendre un rejet, et préparer/convertir des fichiers — sans installation.

Analyse 100 % locale

Vos fichiers (montants, IBAN, noms de tiers…) sont traités uniquement dans votre navigateur. Rien n'est envoyé ni stocké sur nos serveurs — un point clé en environnement bancaire sensible.

Les 6 usages clés

  1. Valider un fichier XML contre les rulebooks SEPA (SCT, SCT Inst, SDD Core/B2B…).
  2. Diagnostiquer un rejet ou lire un relevé (camt).
  3. Convertir des adresses non structurées vers ISO 20022 (échéance nov. 2026). Essai gratuit : 25 adresses.
  4. Générer des fichiers de test conformes (SCT, SDD, ICT, XCT) — gratuit.
  5. Produire de vrais fichiers bancaires transmissibles depuis vos données CSV (SCT, SDD).
  6. Consulter les référentiels : erreurs EBICS, motifs de rejet, balises XML.

Démarrer en 4 étapes

  1. Ouvrez app.ediinsight.app ou l'app iOS — aucune installation.
  2. Utilisez librement les fonctions gratuites : Valider, Comparer, générateur de test, référentiels, significations des codes.
  3. Pour les fonctions payantes : saisissez votre email, recevez un code, tapez-le dans l'app.
  4. Glissez-déposez votre fichier : le résultat s'affiche instantanément.

Guide complet — pas à pas

Chaque fonctionnalité détaillée étape par étape. Les mentions Gratuit et Pro indiquent l'accès requis. Repères : la barre supérieure contient les onglets Accueil, Analyser, Référentiels, EBICS, Plus, et vos initiales (compte) en haut à droite.

Se connecter Gratuit

Connexion sans mot de passe, par code à usage unique — vous restez dans l'app.

  1. Ouvrez une fonction payante (ex. Analyser → Diagnostiquer) : l'écran de connexion s'affiche.
  2. Saisissez votre email, cliquez « Recevoir un code de connexion ».
  3. Ouvrez l'email reçu : il contient un code (6 à 8 chiffres).
  4. Tapez le code dans l'app, cliquez « Se connecter ».
  5. Vous êtes connecté, sur la même page. Vos initiales apparaissent en haut à droite.
Le code remplace tout lien cliquable — insensible aux filtres anti-spam qui « pré-ouvrent » les liens. Il expire au bout d'1 heure ; au besoin, redemandez-en un.

Valider Gratuit Export PDF : Pro

Contrôle la conformité d'un fichier XML contre les rulebooks officiels.

  1. Onglet Analyser → sous-onglet Valider.
  2. Choisissez le Rulebook (SCT, SCT Inst, SDD Core, SDD B2B…) : le référentiel de règles SEPA propre à ce type de flux est appliqué automatiquement au contrôle.
  3. Cliquez « Importer un fichier XML » (ou glissez-déposez le fichier).
  4. Lisez le résultat : soit « conforme / Aucune anomalie détectée », soit la liste des anomalies (balise concernée, règle enfreinte, gravité).
  5. Besoin d'une trace ? Cliquez « Export PDF du rapport » Pro.
  6. Retrouvez vos contrôles via « Historique des validations » (date, fichier, rulebook, compteurs — le contenu n'est pas stocké).
  7. Fichier en erreur ? Cliquez « Fichier KO ? Obtiens le diagnostic complet → » pour basculer vers le Diagnostic.

Diagnostiquer Pro

Explique les rejets et lit les relevés. Formats reconnus : pacs.002, pain.002, erreurs EBICS, et relevés camt.052 / 053 / 054, CFONB 120, MT940.

  1. Onglet Analyser → Diagnostiquer.
  2. Chargez le fichier.
  3. Vue Diagnostic des rejets : chaque opération rejetée devient une carte avec Description, Causes probables, Actions recommandées, le niveau de Confiance et l'Origine.
  4. Si le fichier est un relevé, basculez sur Relevé complet : toutes les écritures s'affichent, avec une recherche (date, montant, tiers, référence).
  5. Exportez les rejets en CSV ou TXT.
« Aucun rejet détecté » = le compte rendu est bon (tout est passé). Utilisez alors la vue Relevé pour l'inventaire complet.

Comparer Gratuit

Compare deux fichiers XML côte à côte — pour vérifier l'impact d'une modification ou détecter l'écart entre deux versions d'un flux.

  1. Onglet Analyser → Comparer.
  2. Chargez le fichier A puis le fichier B.
  3. Les différences structurelles s'affichent : balises ajoutées, modifiées ou supprimées, valeur avant/après.
Cas pratique : vous avez corrigé un IBAN dans un fichier rejeté. Comparer montre exactement ce qui a changé — sans relire 200 lignes de XML.

Pourquoi ? — Contraintes bancaires Pro

Votre fichier est conforme au rulebook SEPA mais la banque le rejette quand même. Chaque établissement a ses propres exigences au-delà de la norme officielle.

  1. Onglet Analyser → Pourquoi ?.
  2. Consultez les contraintes par banque : champs supplémentaires, longueurs réduites, formats spécifiques.
  3. Croisez avec le Diagnostic pour anticiper un refus avant émission.
Cas pratique : rejet AC01 malgré un fichier SCT conforme. « Pourquoi ? » indique que votre banque exige un BIC débiteur explicite — et comment corriger.

Convertisseur d'adresses 25 €/mois · 250 €/an · inclus Pro

Convertit les adresses non structurées (AdrLine) vers le format structuré ISO 20022 (PstlAdr), obligatoire à partir de novembre 2026.

  1. Onglet Analyser → Adresses (ou la tuile de l'Accueil).
  2. Chargez votre fichier : adresse originale à gauche, aperçu converti à droite.
  3. Repérez les statuts : conforme, déjà structuré, champs manquants, ou saisie manuelle requise.
  4. Corrigez les champs manquants directement dans l'aperçu.
  5. Essai gratuit jusqu'à 25 adresses. Au-delà : abonnement (25 €/mois ou 250 €/an, inclus dans le Pro).
  6. Cliquez « Exporter le fichier converti » pour récupérer le XML corrigé + le log d'exclusions. Historique dans « Historique des conversions ».

Générer — fichiers de test Gratuit

Produit des fichiers de test à données fictives (pain.001 / pain.008), pour éprouver vos contrôles.

  1. Onglet Analyser → Générer → Fichier de test.
  2. Type : SCT, SCT Inst, SDD Core, SDD B2B, ICT ou XCT.
  3. Réglez le nombre de transactions, montants et tiers.
  4. Option Défauts à injecter : fichier volontairement non conforme (pour tester vos contrôles).
  5. Téléchargez le XML ou revalidez-le dans Valider.

Générer — depuis mes données (CSV) 20 €/mois · 220 €/an · inclus Pro

Produit de vrais fichiers bancaires transmissibles : virement (SCT) et prélèvement (SDD) à partir de vos données CSV, avec contrôle qualité automatique contre les rulebooks avant téléchargement.

  1. Onglet Analyser → Générer → Depuis mes données.
  2. Chargez votre CSV (bénéficiaires, montants, IBAN, références…) — utilisez le modèle fourni.
  3. Pour le prélèvement : gestion des mandats et regroupement par nature/séquence.
  4. Arbitrez le BtchBookg (comptabilisation globale ou unitaire).
  5. Le fichier est validé automatiquement contre le rulebook avant téléchargement.
  6. Vérifiez montants et IBAN à l'écran de confirmation, puis téléchargez le XML prêt à transmettre.
Vous restez responsable des données saisies et des fichiers transmis à votre banque : le contrôle porte sur la conformité formelle, pas sur l'exactitude métier de vos données.

Référentiels Gratuit

  1. Onglet Référentiels.
  2. Sous-onglets : Services bancaires, Balises XML ISO 20022, EDIFACT.
  3. Parcourez ou recherchez la signification d'une balise, d'un service ou d'un segment.

EBICS & Motifs de rejet Signification gratuite Résolution Pro

  1. Onglet EBICS pour les erreurs 900xx / 91xxx ; les motifs de rejet ISO sont accessibles depuis l'Accueil.
  2. La signification (code, libellé, description) est gratuite.
  3. La résolution (causes, actions, responsable) est réservée aux abonnés Pro.
  4. Retrouvez aussi les versions EBICS (H003 / H004 / H005) et les protocoles (onglet Plus).
